# SKILL.md

Навык для Codex/агента: разработка Android-приложения **Календарь накоплений**.

## Контекст

Пользователь каждый день откладывает сумму, равную номеру дня в году, умноженному на базовую ставку. По умолчанию ставка равна `1 ₽`. Пользователь вручную подтверждает взнос. Приложение отмечает день в календаре, показывает текущий баланс, прогноз до конца года и ежемесячные ободряющие отчёты.

## Ключевые инварианты

1. Подтверждённая запись хранит фактическую сумму на момент подтверждения.
2. Изменение базовой ставки не меняет прошлые записи.
3. Баланс считается только по подтверждённым записям.
4. Пропущенные прошлые дни не добавляются в прогноз автоматически.
5. Отчёт за месяц показывает факт, а не план.
6. Високосный год должен давать 366 дней.
7. Все расчёты денег хранить в `Long`, не в `Float`/`Double`.

## Где что искать

```text
SavingsCalculator.kt       -> чистая бизнес-логика
SavingsEntry.kt            -> Room entity подтверждённого взноса
SavingsDao.kt              -> запросы к базе
SettingsRepository.kt      -> базовая ставка и настройки уведомлений
ReminderScheduler.kt       -> планирование ежедневных напоминаний
MonthlyReportReceiver.kt   -> уведомление последнего дня месяца
SavingsViewModel.kt        -> состояние UI
HomeScreen.kt              -> главный экран
CalendarScreen.kt          -> календарь
HistoryScreen.kt           -> история
SettingsScreen.kt          -> настройки
```

## Типовые задачи

### Добавить новый расчёт

1. Начать с `SavingsCalculator`.
2. Добавить unit-тест.
3. Подключить результат во ViewModel.
4. Отобразить в UI.

### Изменить хранение

1. Добавить миграцию Room.
2. Обновить Entity/Dao.
3. Проверить старые сценарии.
4. Не ломать подтверждённые записи.

### Изменить уведомления

1. Проверить Android permissions.
2. Не выполнять долгую работу в `BroadcastReceiver` без `goAsync`.
3. После изменения времени перепланировать уведомления.
4. Учитывать Android 12+ exact alarm restrictions.

### Добавить экран

1. Создать файл в `ui/screen/<name>`.
2. Не класть расчёты в Composable.
3. Получать данные через ViewModel state.
4. Использовать цвета из `SavingsTheme`.

## Минимальный набор тестов

- `amountForDay(1, 1) == 1`
- `amountForDay(53, 1) == 53`
- `amountForDay(53, 5) == 265`
- обычный год: `66795` при ставке `1`
- високосный год: `67161` при ставке `1`
- прогноз не включает прошлые пропущенные дни;
- месячный отчёт считает только подтверждённые дни выбранного месяца;
- годовой итог считает только подтверждённые дни выбранного года.

## Стиль кода

- Kotlin-first.
- Простые имена.
- Явные типы там, где это улучшает читаемость.
- `LocalDate` для календарных дат.
- Деньги в минимальных единицах текущей валюты или в рублях как `Long` для MVP.
- Без магических чисел в UI.
- Тексты для пользователя на русском.
